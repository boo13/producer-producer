## Directory Purposes

**css/**
- Purpose: CSS stylesheets
- Contains: `styles.css` - all page styling
- Key files: Single stylesheet for entire site
- Subdirectories: None

**images/**
- Purpose: Static image assets
- Contains: `newsletter.jpg` - header illustration
- Key files: One image currently
- Subdirectories: None

**js/**
- Purpose: JavaScript functionality
- Contains: `subscribe-form.js` - form validation and submission
- Key files: Single script for form handling
- Subdirectories: None

**Ref/**
- Purpose: Design reference materials
- Contains: Reference image for retro desktop redesign
- Key files: `Ref1.jpg` - target visual design
- Subdirectories: None

## Key File Locations

**Entry Points:**
- `index.html` - Main (only) HTML page

**Configuration:**
- `CNAME` - GitHub Pages custom domain (producer-producer.com)

**Core Logic:**
- `js/subscribe-form.js` - Form validation, spam protection, submission

**Styling:**
- `css/styles.css` - All visual styling

**Testing:**
- None configured

**Documentation:**
- `README.md` - Project overview
- `CLAUDE.md` - Claude Code instructions
- `PRD.md` - Product requirements for redesign

## Naming Conventions

**Files:**
- kebab-case for multi-word files (`subscribe-form.js`)
- lowercase for single-word files (`styles.css`)
- UPPERCASE for documentation (`README.md`, `CLAUDE.md`)

**Directories:**
- lowercase for all directories
- Singular names (`css`, `js`)
- Exception: `Ref` is capitalized (reference materials)

**Special Patterns:**
- `.md` for documentation
- Single CSS file (no partials/imports)
- Single JS file (no modules)

## Where to Add New Code

**New Feature:**
- Primary code: Modify `index.html` for structure
- Styling: Add to `css/styles.css`
- JavaScript: Add to `js/subscribe-form.js` or create new file
- Tests: Not applicable (no test framework)

**New Component/Module:**
- For the PRD redesign: Create new "window" components in `index.html`
- Add CSS classes to `css/styles.css`
- Minimize new JS (static design per PRD)

**New Assets:**
- Images: `images/`
- Reference materials: `Ref/`
